About the Sandbox

Project Overview

This malware analysis sandbox provides a secure, isolated environment for examining suspicious software, observing how they work, and developing defensive strategies. The system is built using virtualization technology to ensure complete containment of potentially dangerous code.

Our goal is to provide this workstation for ORTSOC students to develop cybersecurity skills relating to malware and defenses and contribute documentation and live videos.

Architecture

Virtualization Layer

Built on VirtualBox, using snapshots for rapid restoration to a clean slate.

Network Isolation

Completely isolated virtual network with optional internet simulation using INetSim https://www.kali.org/tools/inetsim/

Analysis VMs

Windows 10 running with FlareVM for malware analysis tools, and Linux Ubuntu based REMnux, to study on multiple operating systems.

Monitoring Tools

Process Monitor, Wireshark, API monitoring, memory forensics tools, and more.

Safety Protocols

  • All malware samples are executed only in isolated virtual machines
  • No direct network access to host systems or external networks
  • Snapshots in VirtualBox allow instant rollback to clean states
  • All analysis is performed on dedicated hardware, isolated from university network systems
  • Samples are stored with stakeholder, Brandon Ellis
  • Workstation is hosted in a RFID access controlled room (Security Engineering ORTSOC office)

Legal & Ethical Notice

This project is strictly for educational and defensive security research. The execution of malware samples is conducted in accordance with applicable laws and regulations. Getting Started

System Requirements

  • Host system with at least 16GB RAM (32GB recommended)
  • 100GB+ free disk space for VMs and snapshots
  • Intel VT-x or AMD-V virtualization support enabled in BIOS
  • VirtualBox or VMware Workstation

Initial Setup

  1. Download and install your chosen virtualization platform
  2. Install FLARE VM (Windows-based analysis) on top of Windows VM
  3. Deploy REMnux (Linux-based analysis) as it's own Ubunutu based VM
  4. Create snapshot of clean base VMs for easy rollback
  5. Disable network hardware and remove ethernet cord

Tool Documentation

FLARE VM

A fully customizable, Windows-based security distribution for malware analysis, incident response, penetration testing, and reverse engineering.

REMnux

An Ubuntu based Linux toolkit for reverse-engineering and analyzing malicious software. Includes tools for examining browser based threats, malicious documents, and more.

Process Monitor

Advanced monitoring tool for Windows that shows real-time file system, registry, and process/thread activity.

Wireshark

Network protocol analyzer for capturing and examining network traffic generated by malware samples.

How-To Guides

Air-Gapped Malware Sandbox Setup

Complete guide for configuring an isolated malware analysis environment

Overview

Quick Note

Our project used an HP Z230 computer. Based on your own hardware, the BIOS-related steps will likely be different.

Project Description

This guide walks through the configuration of a malware sandbox on an air-gapped computer. It uses AlmaLinux as the host OS with VirtualBox running two analysis VMs: REMnux (Linux-based) and FlareVM (Windows-based).

Host System Configuration

Boot AlmaLinux from USB

  1. Insert USB drive with AlmaLinux
  2. Press F10 during boot to access boot menu
  3. Change boot order to boot from USB first
  4. Boot with AlmaLinux

Install VirtualBox

Download and install VirtualBox from the Oracle Linux website for Oracle Linux 10/Red Hat Enterprise Linux 10 compatibility.

Resolving a "VT-x is disabled in the BIOS for all CPU modes" error

  1. Restart the computer and enter BIOS
  2. Navigate to virtualization settings
  3. Enable VT-x (Intel) or AMD-V (AMD)
  4. Save and exit BIOS

Resolving a "VT-x is Being Used by Another Hypervisor" error

  1. Create a script to disable conflicting hypervisor services: 
    2. nano /home/[username]/vtfix.sh
  2. Make it executable: 
    3. chmod +x /home/[username]/vtfix.sh
  3. Open crontab: 
    4. sudo crontab -e
  4. Set the script to run on boot: 
    5. @reboot /home/[username]/vtfix.sh

Resolving a "Kernel driver errors (rc=-1908)" error

  1. Update your system and install required dependencies: 
    2. sudo dnf update -y && sudo dnf install -y wget kernel-devel kernel-headers gcc make perl elfutils-libelf-devel
  2. To mount USB drives with Windows file systems (loading malware from external storage): 
    3. sudo dnf install exfat-fuse exfat-utils ntfs-3g

REMnux VM Setup

NOTE: REMnux is its own pre-configured VM, not a toolkit you install on Ubuntu.

Download and Import REMnux

Download:

  • Visit REMnux.org → Navigate to Get REMnux → Download REMnux Virtual Appliance (.ova file)

Import into VirtualBox:

  • File → Import Appliance → Select .ova file → Import

Configure REMnux VM

The VM comes preconfigured with analysis tools. Set default credentials and create a snapshot after successful import.

FlareVM Setup

Prepare Windows 10 VM

Create a Windows 10 VM in VirtualBox with these minimum specifications:

  • System Settings:
    • Base Memory: 4096 MB minimum (8 GB preferred)
    • Processors: 2 minimum (4 preferred)
    • Enable PAE/NX
    • Enable Nested VT-x/AMD-V if available
    • Set boot order to hard disk first
  • Storage:
    • Allocate at least 60 GB of disk space (FlareVM requires substantial storage)
  • Network:
    • Adapter 1: NAT or Bridged (will be disabled later for air-gapping)

Install Windows 10

  1. Start the VM and go through the Windows Setup process (may take a little while).
  2. Connect to the internt during setup to install updates.
  3. Ensure you have admin rights.

Disable Windows Defender

FlareVM requires Windows Defender to be disabled:

  • Go to Windows Settings → Update & Security → Windows Security → Virus & Threat Protection → Manage Settings → Turn off Real-time Protection
  • Keep UAC enabled (FlareVM expects it).
  • If Windows Defender won't disable through settings, use Group Policy:
    • Open Local Group Policy Editor → Computer Configuration → Administrative Templates → Windows Components → Windows Defender Antivirus → Turn off Windows Defender Antivirus → Enabled
    • Run gpupdate /Force and reboot.
    • NOTE: Having followed these steps, if FlareVM still gives a warning that Windows Defender is not disabled later, proceed with installation anyways.
Create a snapshot in Virtualbox before proceeding.

Install FlareVM

  1. Open Powershell as Administrator and change the execution policy: 
    Set-ExecutionPolicy Unrestricted -Force
  2. Download and run the FlareVM installer: 
    Invoke-WebRequest https://raw.githubusercontent.com/mandiant/flare-vm/master/install.ps1 -UseBasicParsing | iex
  3. During installation, you can select default options or customize tools (PuTTY and CapeSandbox are two we chose to include).
  4. The installation process will take considerable time and require multiple reboots. Create a VirtualBox snapshot after successful installation.

Air-Gapping the System

Disable Network on Host Machine

Access BIOS and disable networking.

Disable Intel AMT (if present)

  • Press CTRL+P during boot to enter Intel MEBx menu
  • Login with admin credentials (default: admin)
  • Set/confirm new password: Password123!
  • Unconfigure network access
  • Go to BIOS → Advanced → Management Operations → AMT → Disabled

Disable Network Controller

  • BIOS → Security → Device Security → Network Controller → Device Hidden.

This ensures the physical machine cannot connect to any network.

Malware Transfer

Setting up Shared Folder for REMnux

  1. In VirtualBox, go to VM Settings → Shared Folders → Add Share
  2. Create a folder on your host.
  3. Set folder path in VirtualBox to the malware folder location
  4. Set mount point to desired location on folder on VM (e.g. /home/remnux/Documents/Malware).
  5. Enable Read-only and Auto-mount.

Setting up Shared Folder for FlareVM:

FlareVM uses a different approach, direct USB mounting rather than shared folders.

  1. Install VirtualBox Extension Pack:

    1. Put the VirtualBox Extension Pack (Oracle VM Extension 7.0.6) on a USB drive
    2. Mount the USB on the host system
    3. Install the extension pack from the USB

  2. Configure User Permissions:

    1. Add your user to the vboxusers group to enable USB passthrough:
      2. sudo usermod -aG vboxusers team10
    2. Restart the system for the group changes to take effect

  3. Enable USB in FlareVM:

    1. Open VirtualBox
    2. Select the FlareVM virtual machine
    3. Go to Settings → USB
    4. Set USB controller to USB 3.0
    5. Add USB device filters to mount your USB drive

    4. This allows the FlareVM to access malware samples directly from the USB drive.

Safe Malware Transfer Workflow

  1. Transfer malware files to the VM using the shared folder
  2. Copy malware from shared folder to a local folder within the VM
  3. Immediately disable/delete the shared folder in VirtualBox settings
  4. Now you have complete isolation with no connection between host and VM
