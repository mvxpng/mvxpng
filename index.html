About the Sandbox

Project Overview

This malware analysis sandbox provides a secure, isolated environment for examining suspicious software, observing how they work, and developing defensive strategies. The system is built using virtualization technology to ensure complete containment of potentially dangerous code.

Our goal is to provide this workstation for ORTSOC students to develop cybersecurity skills relating to malware and defenses and contribute documentation and live videos.

Architecture

Virtualization Layer

Built on VirtualBox, using snapshots for rapid restoration to a clean slate.

Network Isolation

Completely isolated virtual network with optional internet simulation using INetSim https://www.kali.org/tools/inetsim/

Analysis VMs

Windows 10 running with FlareVM for malware analysis tools, and Linux Ubuntu based REMnux, to study on multiple operating systems.

Monitoring Tools

Process Monitor, Wireshark, API monitoring, memory forensics tools, and more.

Safety Protocols

  • All malware samples are executed only in isolated virtual machines
  • No direct network access to host systems or external networks
  • Snapshots in VirtualBox allow instant rollback to clean states
  • All analysis is performed on dedicated hardware, isolated from university network systems
  • Samples are stored with stakeholder, Brandon Ellis
  • Workstation is hosted in a RFID access controlled room (Security Engineering ORTSOC office)

Getting Started

System Requirements

  • Host system with at least 16GB RAM (32GB recommended)
  • 100GB+ free disk space for VMs and snapshots
  • Intel VT-x or AMD-V virtualization support enabled in BIOS
  • VirtualBox or VMware Workstation

Initial Setup

  1. Download and install your chosen virtualization platform
  2. Install FLARE VM (Windows-based analysis) on top of Windows VM
  3. Deploy REMnux (Linux-based analysis) as it's own Ubunutu based VM
  4. Create snapshot of clean base VMs for easy rollback
  5. Disable network hardware and remove ethernet cord

Tool Documentation

FLARE VM

A fully customizable, Windows-based security distribution for malware analysis, incident response, penetration testing, and reverse engineering.

REMnux

An Ubuntu based Linux toolkit for reverse-engineering and analyzing malicious software. Includes tools for examining browser based threats, malicious documents, and more.

Process Monitor

Advanced monitoring tool for Windows that shows real-time file system, registry, and process/thread activity.

Wireshark

Network protocol analyzer for capturing and examining network traffic generated by malware samples.

Configuring Network Isolation

Step-by-step network setup for safe malware execution

Step 1: Create Virtual Network

In VirtualBox, go to File → Host Network Manager. Create a new host-only network with DHCP disabled.

Step 2: Configure VM Network Settings

Set your analysis VM's network adapter to "Host-only Adapter" and select the network you created.

Step 3: Verify Isolation

Boot the VM and attempt to ping external addresses. Confirm no external connectivity exists.

Step 4: Optional Internet Simulation

Install and configure INetSim on a separate VM to simulate internet services for malware that requires connectivity.
